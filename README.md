# MAX ↔ Telegram мост

Пересылает сообщения вашего личного аккаунта **MAX** (max.ru) в **Telegram**
через бесплатного бота и позволяет **отвечать прямо из Telegram**. Работает на
этом ПК, ничего не стоит.

## Что умеет

- **MAX → Telegram:** текст, фото, стикеры, голосовые/аудио, **файлы и видео**
  пересылаются содержимым. Файлы/видео скачиваются из MAX и загружаются в
  Telegram (ограничение Telegram — 50 МБ на загрузку; что больше, приходит
  уведомлением «открыть в MAX»).
- **Telegram → MAX (ответы):** чтобы ответить, сделайте в Telegram **«Ответить»
  (Reply / свайп по сообщению)** на пересланном сообщении и напишите текст — он
  уйдёт в тот же чат MAX. Если отправить в теме Telegram файл/фото/стикер, мост
  скачает его через Telegram Bot API и загрузит в MAX как файл-вложение.

## Как запустить

Дважды кликните `run.bat` (или выполните `venv\Scripts\python.exe main.py`).

При первом запуске мастер настройки попросит:

1. **Токен Telegram-бота** — создайте бота у [@BotFather](https://t.me/BotFather)
   командой `/newbot` (бесплатно, 2 минуты).
2. **Отправить `/start` вашему боту** — так скрипт узнает, куда слать сообщения.
3. **Токен MAX из веб-версии** — прямой вход по SMS закрыт капчей MAX,
   поэтому токен берётся из веб-клиента:
   - Войдите на [web.max.ru](https://web.max.ru) (номер, капча, код — как обычно).
   - Нажмите `F12` → вкладка **Console** → вставьте строку и `Enter`:
     ```
     copy(JSON.parse(localStorage.__oneme_auth).token)
     ```
   - Токен скопируется в буфер; вставьте его в окно мастера.

   Токен сохраняется, повторно входить не нужно. **Не выходите (Logout)
   из web.max.ru** — это аннулирует токен; просто закройте вкладку.

После настройки скрипт просто работает: каждое входящее сообщение в MAX
приходит вам в Telegram в виде `MAX | Имя (чат N): текст`.

## Режим «темы как чаты» в Telegram

Чтобы сообщения выглядели как отдельные диалоги, создайте Telegram-группу,
включите в ней **Темы** и добавьте бота администратором с правом управлять
темами. После этого добавьте в `config.json`:

```json
{
  "telegram_topics_enabled": true,
  "telegram_forum_chat_id": -1001234567890,
  "telegram_fallback_chat_id": 2097379459,
  "telegram_preload_topics": true,
  "telegram_seed_last_messages": true,
  "telegram_preload_chat_count": 100
}
```

`telegram_forum_chat_id` — id супергруппы с темами. При первом сообщении из
каждого MAX-чата мост создаст отдельную тему и запомнит связь в `state.json`.
Если тему создать не удалось или `telegram_forum_chat_id` не задан, сообщение
уйдёт старым способом в `telegram_fallback_chat_id`.

`telegram_preload_topics` заранее создаёт темы для последних MAX-чатов при старте.
`telegram_seed_last_messages` один раз отправляет последнее сообщение в каждую тему,
чтобы список тем выглядел как обычные диалоги, а не как лента системных сообщений
«Тема создана». `telegram_preload_chat_count` ограничивает количество чатов.

Внутри темы можно просто написать текст — он уйдёт в соответствующий MAX-чат.
Reply на конкретное сообщение тоже поддерживается и отправляет ответ цитатой.

## Автозапуск при включении ПК (по желанию)

Нажмите `Win+R` → `shell:startup` → скопируйте туда ярлык на `run.bat`.

## Запуск на сервере 24/7

См. [DEPLOY.md](DEPLOY.md) — бесплатные варианты (своё всегда-включённое
устройство через systemd, либо Oracle Cloud Always Free через Docker). Токены
на сервере задаются переменными окружения (`.env`), браузер не нужен.

## Важно знать

- Используется **неофициальная** библиотека [vkmax](https://github.com/nsdkinx/vkmax)
  (MAX не даёт официального API для личных аккаунтов). Теоретически MAX может
  ограничить такую сессию — она выглядит для него как вход через веб-версию.
- Токены хранятся открытым текстом в `config.json` — не пересылайте этот файл.
- Если MAX «разлогинил» сессию (или вы вышли из web.max.ru), удалите
  `config.json` и пройдите настройку заново, взяв свежий токен.
- Логи пишутся в `bridge.log`.

## Ограничения

- **Размер файла/видео** для пересылки — до ~50 МБ (лимит Telegram для ботов).
  Что больше — приходит уведомлением с именем/размером, открывается в MAX.
- Ссылки на файлы/видео в MAX временные (~24 ч), но это не важно: мост
  скачивает их сразу при получении сообщения.
- Ответом из Telegram можно отправить **текст и файлы**. Стикеры Telegram
  отправляются в MAX как файл-вложение (`.webp`, `.webm` или `.tgs`), потому что
  нативные stickerId у Telegram и MAX несовместимы.
- Соответствие «сообщение Telegram → чат MAX» хранится в памяти и сбрасывается
  при перезапуске моста: на сообщения, пересланные до перезапуска, ответить
  через Reply уже не получится (на новые — да).
- Неизвестные типы вложений логируются в `attaches.log` — это помогает потом
  добавить их поддержку.

## Состав

| Файл | Назначение |
|------|------------|
| `main.py` | точка входа: настройка + запуск моста |
| `setup_wizard.py` | мастер первой настройки |
| `bridge.py` | слушает MAX, пересылает в Telegram, принимает ответы |
| `attaches.py` | разбор вложений MAX (фото, стикеры, файлы, …) |
| `mediamax.py` | получение ссылок на файлы/видео из MAX (opcode 88/83) |
| `tg.py` | мини-клиент Telegram Bot API (текст + медиа) |
| `config.py` | конфиг из `config.json` или env-переменных |
| `requirements.txt` / `Dockerfile` / `docker-compose.yml` | для сервера |
| `max2tg.service` | systemd-юнит для Linux/Raspberry Pi |
| `DEPLOY.md` | инструкция по бесплатному хостингу |
